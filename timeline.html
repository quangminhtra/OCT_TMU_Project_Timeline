Week 3: September 15th - September 21st, 2025 - Fundamentals Research

  • Researched SIM-to-microcontroller integration methods.

The figure illustrates the data flow of the Outdoor Cat Tracker system. The GPS module captures the cat's location and sends the data to Firebase, which acts as the central database and API hub. The React Native app listens for real-time updates from Firebase and displays the location information on the front-end interface, allowing live tracking and interaction.

Mobile Framework: React Native

  • Enables cross-platform development (iOS & Android) from a single JavaScript codebase.
  • Offers native-like performance and a strong open-source community.

Database & API Solution: Firebase (Firestore + FCM)

  • Provides real-time location updates and push notifications.
  • Supports geofencing, offline access, and seamless integration with React Native SDKs.

  • Investigated app components (dashboard, location) and hardware-software communication methods.

System Entities and User Flow

Main Entities

  • User: Account holders who manage pets and devices.
  • Product Code Database: Stores unique device codes for pairing.
  • Users Database: Stores account credentials and login details.
  • Pets Database: Stores pet profiles and tracking configurations.

Flow Steps

  1. User enters a Product Code (unique device ID).
  2. If incorrect: shows Error; if pairing fails: retry or error.
  3. On success: proceed to Account Creation (username & password).
  4. User provides name, date of birth, email, and phone number.
  5. User adds Pet Details (multiple pets supported).
  6. Setup Geofencing: define safe zones and alerts.
  7. Access the Dashboard for live tracking, alerts, and feedback.
  8. User can log out or continue monitoring.

Key Notes from the Diagram

  • One user can manage multiple product codes (trackers).
  • Each user can link several pets to their account.
  • Each pet can have multiple detection points, but only one is active at a time.
