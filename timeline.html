System Entities and User Flow

Main Entities

  • User: Account holders who manage pets and devices.
  • Product Code Database: Stores unique device codes for pairing.
  • Users Database: Stores account credentials and login details.
  • Pets Database: Stores pet profiles and tracking configurations.

Flow Steps

  1. User enters a Product Code (unique device ID).
  2. If incorrect: shows Error; if pairing fails: retry or error.
  3. On success: proceed to Account Creation (username & password).
  4. User provides name, date of birth, email, and phone number.
  5. User adds Pet Details (multiple pets supported).
  6. Setup Geofencing: define safe zones and alerts.
  7. Access the Dashboard for live tracking, alerts, and feedback.
  8. User can log out or continue monitoring.

Notes

  • One user can manage multiple product codes (trackers).
  • Each user can link several pets to their account.
  • Each pet can have multiple detection points, but only one is active at a time.

Week 10,11: November 3rd - November 23rd, 2025 - Full-Scale Integration & Prototype Finalization (End of Fall Term)

  • Successfully interfaced the Quectel L80-R GPS module to parse NMEA 0183 messages using the TinyGPS++ library, enabling precise coordinate extraction for real-time tracking.
  • Orchestrated a successful Point-to-Point (P2P) radio test between RAK3172 LoRa transceivers, validating the transmission and reception of hexadecimal data packets across the long-range link.
  • Finalized the mobile power solution by integrating an 800mAh rechargeable LiPo battery with an MCP1700T LDO voltage regulator to ensure a stable 3.3V supply for all hardware components.
  • Completed a 100% interactive Figma prototype with comprehensive screen-to-screen flows, meeting WCAG 2.1 AA accessibility standards for contrast, touch targets, and navigation
  • Defined the system’s RESTful API schema (GET/POST) and configured the Raspberry Pi Pico 2 W gateway to bridge incoming radio telemetry with the backend server via HTTP requests

Successfully interfaced with the LoRa Module (i.e., send/receive commands, changed settings, ran Arduino Example program) in the Arduino IDE.

Raspberry Pi Pico 2 W microcontrollers have been ordered and will be used in the receiver module. These microcontrollers feature integrated 2.4GHz Wi-Fi connectivity, enabling wireless transmission of data to the server. The pinout diagram for the Pico 2 W is provided below.

The system API is structured as follows. POST requests include GPS data transmissions from the microcontroller to the server, low-battery status notifications, and new account information submitted by the front-end interface. GET requests are used to retrieve GPS location updates and battery percentage data from the server for display on the front end.
