Capstone Technical Specification

A LoRa-based outdoor cat tracking system built from tracker hardware to mobile software.

Outdoor Cat Tracker combines a GPS-equipped wearable tag, a home-base relay, backend ingestion, and a cross-platform mobile application. The system is designed around one goal: giving owners clearer visibility and geofence-based awareness without putting a cellular subscription on the collar itself.

System Flow

How the system works

The system is built as a simple chain: the collar gathers location data, LoRa carries it back home over a low-power long-range link, the home base forwards it to the backend, and the app turns that data into something an owner can understand quickly.

Tracker

The collar unit reads GPS coordinates, packages the data, and keeps the electronics small enough to stay wearable on the cat.

LoRa Link

LoRa was chosen because it reaches farther than short-range consumer trackers while staying lightweight and low power for a prototype collar.

Home Base

The home receiver listens for packets near the house, then forwards valid pings through Wi-Fi so the rest of the system can process them.

Mobile App

The app shows the latest position, safe-zone state, and alert history in a format that feels useful to owners rather than raw telemetry.

Approach to the project

Where this project fits

Main product idea

This project focuses on localized outdoor safety rather than nationwide cellular coverage. Its strength is a home-centered long-range model that gives owners visibility around the home and nearby area without putting a subscription-powered modem on the collar.

Why it stands out

  • Dedicated long-range local link through LoRa instead of short-range Bluetooth alone
  • Owner-controlled home base instead of reliance on a crowd network of nearby phones
  • Integrated safe-zone workflow with live tracking and alert history
  • No tracker-side cellular subscription in the current prototype architecture

Current tradeoffs

  • Coverage is centered around the installed home base
  • Battery life is still prototype-scale rather than a final commercial runtime target
  • Testing was largely manual and not yet backed by a full automated validation suite
  • Late-stage custom-board reliability issues still separate the prototype from production readiness

Hardware

Hardware specification and why each part was chosen

Tracker-side choices

  • RAK3172: combines embedded control with long-range LoRa communication in a compact module.
  • Quectel L80-R: provides outdoor GPS acquisition while fitting the tracker's power and size limits.
  • Ceramic patch antenna: chosen over a long whip antenna because it better suits a wearable cat-mounted form factor.
  • 600 mAh Li-Po battery: supports untethered field testing with a lightweight battery pack.
  • TP4056 + MCP1700: handle charging and voltage regulation to keep the electronics operating safely and reliably.

Home-base choices

  • RAK3172 receiver: listens continuously for incoming LoRa packets from the tracker tag.
  • Raspberry Pi Pico 2 W: provides the Wi-Fi relay needed to upload incoming tracker data to the server.
  • Stationary USB-powered design: avoids the battery limits of the collar device and supports fixed deployment near the home.
  • PCB and enclosure iterations: move the prototype beyond breadboard development into a more realistic capstone implementation.

Prototype pricing

The full report contains the detailed bill of materials. In summary, the tracker bill of materials is approximately $30.61 CAD and the home base bill of materials is approximately $37.22 CAD.

Software

Software specification and why the stack was chosen

React Native was used to build one mobile codebase across iOS and Android.

Expo accelerated development, testing, and packaging during the capstone build.

Firebase supported authentication and cloud-backed application data management.

Flask was used to receive tracker pings and serve device-specific data to the app.

Application behavior

  • The app is organized around live location, alerts, settings, and pet management.
  • Owners can see the home marker, safe-zone circle, current pet position, and alert history in one workflow.
  • Device verification and active-pet selection support multi-device and multi-pet ownership flows.

Accessibility and UX

  • The design intent follows WCAG 2.1 AA guidance for contrast, touch targets, readability, and clear navigation.
  • The software turns raw location updates into a more understandable safety workflow for non-technical users.
  • Alerting and safe-zone presentation were prioritized to reduce confusion when a cat leaves or returns home.
